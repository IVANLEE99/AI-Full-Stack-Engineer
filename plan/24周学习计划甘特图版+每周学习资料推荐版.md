# 24周学习计划甘特图版 + 每周学习资料推荐版

> 目标：24 周完成从前端到 Go AI 全栈工程师的系统转型。  
> 方法：围绕一个持续演进的主项目推进，每周聚焦一个主题，每月完成一个可展示成果。

---

## 入门策略（根据实践建议调整）

> Go 入门只需掌握 6 件套即可开始做项目，不要等基础全学完再动手：
>
> 1. Go 基础语法 → 2. Gin Web 开发 → 3. GORM / 数据库 → 4. 分层架构 / 规范 → 5. 并发 / 异步任务 → 6. Redis / 缓存
>
> **核心原则：边学边做，先把小项目搞起来。** 第 3 周起就动手写真实小项目，在项目里补语法和工程能力，而不是先啃完所有知识点。

## 二、每周学习资料推荐版

### 第1周：Go基础语法
**学习目标**
- 建立 Go 强类型与显式错误处理思维

**推荐资料**
- Go 官方 Tour
- Go by Example
- 《Go语言圣经》前 1-5 章

**实战任务**
- 用 Go 重写一个你熟悉的前端工具函数模块
- 完成 Todo API CRUD（内存版）

**验收标准**
- 能解释 `if err != nil`
- 能写基础 REST API
- 能读懂 AI 生成的 Go 代码并手改

---

### 第2周：Gin Web开发
**学习目标**
- 能用 Gin 写标准 API

**推荐资料**
- Gin 官方文档
- validator 官方示例
- JWT 最佳实践文章

**实战任务**
- 用 Gin 重写 Todo API
- 加用户注册、登录、JWT 鉴权
- 加鉴权中间件与用户信息接口

**验收标准**
- 能解释中间件执行顺序
- 注册/登录/鉴权链路完整

---

### 第3周：GORM / 数据库（开始第一个小项目）
**学习目标**
- 打通 Go + 数据库，把数据真正落库

**推荐资料**
- GORM 官方文档
- PostgreSQL 或 MySQL 基础教程

**实战任务**
- 选定第一个小项目（Todo 进阶 / 商品后台 / 博客）
- 设计表结构，用 GORM 做模型映射
- 把核心数据从内存改为数据库存储

**验收标准**
- 会建模、会迁移
- 能完成 Go + DB 联调
- 能讲清一次请求的数据流转

---

### 第4周：分层架构 / 规范
**学习目标**
- 把小项目从 demo 升级为工程

**推荐资料**
- Go Project Layout
- Clean Architecture 入门文章

**实战任务**
- 把小项目重构为 Controller / Service / Repository 分层
- 建立统一响应结构与错误码
- 沉淀一个可复用后台骨架

**验收标准**
- 项目分层清晰、职责分明
- 骨架可复用于后续项目

---

### 第5周：并发 / 异步任务
**学习目标**
- 掌握 Go 并发核心能力，并用进小项目

**推荐资料**
- Go by Example 并发章节
- Go 官方博客：Concurrency Patterns
- Go `context` 官方文档

**实战任务**
- 给小项目加异步能力：批量导入 / 通知发送 / 并发拉取
- 用 `channel` + `WaitGroup` 做多协程汇总

**验收标准**
- 会用 `channel`
- 不写并发写 map 事故代码
- 异步任务有基本的超时和错误处理

---

### 第6周：Redis / 缓存
**学习目标**
- 学会给接口接缓存

**推荐资料**
- Redis 官方文档
- go-redis 文档
- 缓存穿透/击穿/雪崩专题文章

**实战任务**
- 给小项目关键接口接入 Redis 缓存
- 设计缓存读写与失效策略

**验收标准**
- 能讲清缓存命中/失效流程
- 接口响应有可见优化

---

### 第7周：小项目冲刺
**学习目标**
- 综合运用入门 6 件套，把小项目做扎实

**推荐资料**
- GORM 事务文档
- 你所选项目对应的业务参考资料

**实战任务**
- 选定真实小项目（订单 / 博客 / 门店后台）
- 补全核心业务与事务
- 关键写操作用事务保证一致性

**验收标准**
- 核心业务链路跑通
- 关键写操作有事务保护

---

### 第8周：第一个可展示项目
**学习目标**
- 完成并演示第一个完整小项目

**推荐资料**
- Swagger / OpenAPI 基础资料
- Postman 或 Apifox 使用教程

**实战任务**
- 收尾、联调、写 API 文档
- Git 规范提交
- 完成“注册 → 登录 → 核心业务 → 查询”闭环

**验收标准**
- 项目可完整演示
- 接口能跑通
- 代码结构成型，可对外展示

---

### 第9周：缓存与并发进阶
**学习目标**
- 学会做抗压接口，掌握可控的大批量任务处理

**推荐资料**
- Redis 官方文档、go-redis 文档
- 缓存穿透/击穿/雪崩专题文章
- Go `context` 官方文档、协程池库 `ants` 资料

**实战任务**
- 给已有小项目做缓存加固（穿透/雪崩应对）
- 实现批量处理任务模块，用 Worker Pool 控制并发数
- 支持超时取消

**验收标准**
- 缓存失效逻辑完整
- 1 万条任务可稳定处理，不滥开 goroutine，有超时和失败处理
