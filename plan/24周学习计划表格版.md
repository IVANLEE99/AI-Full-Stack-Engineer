# 24周学习计划表格版

> 目标：在 24 周内，从前端工程师转型为具备 Go 后端、AI 集成、Agent 编排与企业级交付能力的 AI 全栈工程师。  
> 主线：前 8 周打 Go 地基，9-16 周补生产级工程能力，17-24 周进入 AI / Agent / RAG / 企业级交付。

---

## 24周学习计划总表

| 周次 | 主题 | 学习重点 | 实战任务 | 周产出 / 验收标准 |
|---|---|---|---|---|
| 第1周 | Go 语法入门 | `struct`、`interface`、`slice`、`map`、`error`、`defer`、`go mod` | 用 Go 重写一个简单工具模块；做一个 Todo API 基础 CRUD | 能独立写出基础 REST API；理解 `if err != nil`；本地可跑通 |
| 第2周 | Gin Web 基础 | Gin 路由、中间件、参数绑定、校验、`Context`、JWT 基础 | 做用户注册/登录接口；实现鉴权中间件 | 用户注册、登录、鉴权链路完整；能解释中间件执行顺序 |
| 第3周 | 数据库与 ORM | GORM、模型设计、基础 SQL、Redis 基础、JSON 序列化 | 做商品/分类/库存模块；实现 CRUD + 查询 | 能完成 Go + DB 基础联调；模型、表结构、接口对应清晰 |
| 第4周 | 项目分层架构 | `cmd/`、`internal/`、Controller/Service/Repository、依赖注入思路 | 搭建一个 CMS Admin 或基础后台模板 | 形成可复用项目骨架；目录结构清晰，职责分层合理 |
| 第5周 | 鉴权与规范化重构 | JWT、权限中间件、用户上下文、Clean Architecture 基础 | 重构登录与用户模块，统一错误码和返回结构 | 项目具备登录、鉴权、统一响应；代码开始具备企业规范 |
| 第6周 | 事务与核心业务 | 事务、乐观锁、悲观锁、订单/库存业务建模 | 实现下单、扣库存、防超卖 | 关键业务链路正确；并发场景下不会出现明显超卖问题 |
| 第7周 | Go 并发基础 | `goroutine`、`channel`、`select`、`WaitGroup`、异步任务思想 | 做并发下载器或通知发送模块 | 能解释 Go 并发模型；写出基本异步处理逻辑 |
| 第8周 | 第一阶段综合项目 | 串联 Gin + GORM + JWT + 并发 + CRUD | 完成“注册→登录→下单→查询订单”完整闭环 | 第一个完整 Go 项目可演示；接口能跑通；代码结构基本成型 |
| 第9周 | Redis 实战 | 缓存穿透、雪崩、TTL、布隆过滤器思路、ZSet/List 用法 | 给用户信息或商品详情接口加缓存 | 能说明缓存策略；接口响应明显优化；缓存失效逻辑完整 |
| 第10周 | 并发进阶与工作池 | Worker Pool、协程池、`context` 超时取消、批处理 | 实现批量处理任务模块，控制并发数 | 1万条任务可稳定处理；能控制超时和取消 |
| 第11周 | Docker 容器化 | Dockerfile、多阶段构建、`docker-compose` | 容器化 Go 服务 + MySQL + Redis | 本地能一键启动完整环境；镜像结构合理 |
| 第12周 | 配置与日志 | Viper、环境配置、结构化日志、`trace_id` | 给项目接入配置文件和 JSON 日志 | 项目支持 dev/prod 配置切换；日志可用于排障 |
| 第13周 | 单元测试与 Mock | 表驱动测试、`gomock`、Service 层测试、边界场景 | 为核心业务逻辑补测试 | 核心服务测试覆盖率达到约 70%-80%；能模拟外部依赖 |
| 第14周 | 联调与自动化测试 | 接口测试、前后端联调、Playwright 冒烟测试 | 为关键流程写端到端测试 | 能自动验证核心业务主链路；前后端联调顺畅 |
| 第15周 | 安全与性能分析 | SQL 注入、越权、未授权访问、`pprof`、慢 SQL | 对已有项目做一次安全审计和性能检查 | 输出一份安全/性能问题清单，并修复关键项 |
| 第16周 | CI/CD 与压测 | GitHub Actions / GitLab CI、构建、测试、打包、压测 | 配置自动化流水线并做接口压测 | Push 后能自动测试构建；有一份压测报告和瓶颈结论 |
| 第17周 | SSE 与流式接口 | `http.Client`、SSE、流式转发、`Flusher`、超时处理 | 实现 `/chat/stream` 接口 | 能把 LLM 输出流式返回前端；前端可见打字机效果 |
| 第18周 | Prompt 与结构化输出 | Prompt 角色设计、JSON Mode、Function Calling | 做“自然语言转结构化 JSON”接口 | 模型输出稳定可解析；接口可供前端/服务直接消费 |
| 第19周 | RAG 与向量检索 | Embedding、Chunk、向量库、PGVector/Milvus、召回 | 做一个企业 FAQ / 制度问答 Bot | 能完成文档入库、检索、问答；回答能引用知识内容 |
| 第20周 | Tool Calling 与 Agent 基础 | Tool Schema、Tool Call 回填、ReAct 思路、MCP 基础认知 | 做一个能查天气/订单/数据库的简易 Agent | Agent 能调用工具完成任务；具备基础工作流能力 |
| 第21周 | 多 Agent 与业务拆解 | 主管 Agent / 子 Agent、任务分解、PRD 到技术任务映射 | 选一个真实业务需求，拆成 PRD、架构、开发任务 | 能从一句话需求拆到结构化交付；开始具备“AI Manager”思维 |
| 第22周 | Skill 化与知识沉淀 | Prompt 模板、规则文件、工作流标准化、可复用知识库 | 整理一套 Go 微服务生成规范、审查规则、AI 协作模板 | 形成团队可复用的 Skill 文档 / 知识库资产 |
| 第23周 | 企业级 AI Agent 项目开发 | Agent 编排、工具集成、静态分析/业务规则接入 | 开发一个 AI 代码审查 Agent 或数据分析 Agent | 主链路跑通；能接收输入、调用工具、产出结果 |
| 第24周 | 上线前加固与毕业项目 | RBAC、审计日志、限流、重试、降级、部署上线 | 将第23周项目部署到测试环境并完成演示 | 一个可演示、可审计、可部署的 AI Native 项目正式成型 |
