### 1. Goroutine 基础（星期一）

**核心概念**
- Goroutine 是什么：Go 的轻量级线程
- 如何启动 goroutine：`go` 关键字
- Goroutine 的调度模型：GMP 模型基础认知
- Goroutine 与线程的区别

**代码示例**
```go
package main

import (
    "fmt"
    "time"
)

func sayHello(name string) {
    for i := 0; i < 3; i++ {
        fmt.Printf("Hello from %s: %d\n", name, i)
        time.Sleep(100 * time.Millisecond)
    }
}

func main() {
    // 启动多个 goroutine
    go sayHello("goroutine-1")
    go sayHello("goroutine-2")
    go sayHello("goroutine-3")
    
    // 主 goroutine 等待
    time.Sleep(1 * time.Second)
    fmt.Println("Main function exits")
}
```

### 2. Channel 通信（星期一-星期二）

**核心概念**
- Channel 是 goroutine 之间通信的管道
- 无缓冲 channel vs 有缓冲 channel
- Channel 的发送和接收操作
- Channel 的关闭

**代码示例**
```go
package main

import "fmt"

// 无缓冲 channel
func withoutBuffer() {
    ch := make(chan int)
    
    go func() {
        ch <- 42  // 发送数据
        fmt.Println("Sent 42")
    }()
    
    val := <-ch  // 接收数据
    fmt.Println("Received:", val)
}

// 有缓冲 channel
func withBuffer() {
    ch := make(chan int, 2)  // 缓冲区大小为 2
    
    ch <- 1
    ch <- 2
    // 不会阻塞，因为缓冲区未满
    
    fmt.Println(<-ch)
    fmt.Println(<-ch)
}

// 生产者-消费者模式
func producerConsumer() {
    ch := make(chan int, 5)
    
    // 生产者
    go func() {
        for i := 0; i < 10; i++ {
            ch <- i
            fmt.Println("Produced:", i)
        }
        close(ch)  // 生产完毕，关闭 channel
    }()
    
    // 消费者
    for val := range ch {  // range 会在 channel 关闭时退出
        fmt.Println("Consumed:", val)
    }
}

func main() {
    withoutBuffer()
    withBuffer()
    producerConsumer()
}
```

- [ ] 实现一个数据处理管道：读取 → 处理 → 输出
- [ ] 用 channel 实现多个 goroutine 的结果汇总

### 3. Select 多路复用（星期二）

**核心概念**
- select 语句用于同时等待多个 channel 操作
- select 的随机性：多个 case 同时就绪时随机选择
- default 分支：非阻塞操作

**代码示例**
```go
package main

import (
    "fmt"
    "time"
)

func worker(id int, ch chan string) {
    time.Sleep(time.Duration(id) * time.Second)
    ch <- fmt.Sprintf("Worker %d done", id)
}

func main() {
    ch1 := make(chan string)
    ch2 := make(chan string)
    
    go worker(1, ch1)
    go worker(2, ch2)
    
    for i := 0; i < 2; i++ {
        select {
        case msg1 := <-ch1:
            fmt.Println("From ch1:", msg1)
        case msg2 := <-ch2:
            fmt.Println("From ch2:", msg2)
        case <-time.After(3 * time.Second):
            fmt.Println("Timeout!")
        }
    }
}

// 非阻塞操作示例
func nonBlocking() {
    ch := make(chan int)
    
    select {
    case val := <-ch:
        fmt.Println("Received:", val)
    default:
        fmt.Println("No data available")
    }
}
```

**实践任务**
- [ ] 实现超时控制：3 秒内未完成则放弃
- [ ] 用 select 实现优先级队列（尝试从高优先级 channel 读取）
- [ ] 实现一个可取消的任务

### 4. WaitGroup 同步（星期三）

**核心概念**
- WaitGroup 用于等待一组 goroutine 完成
- Add()、Done()、Wait() 三个核心方法
- 常见错误：Add 数量不匹配、Wait 在 Add 之前

**代码示例**
```go
package main

import (
    "fmt"
    "sync"
    "time"
)

func task(id int, wg *sync.WaitGroup) {
    defer wg.Done()  // 函数退出时调用
    
    fmt.Printf("Task %d starting\n", id)
    time.Sleep(time.Second)
    fmt.Printf("Task %d done\n", id)
}

func main() {
    var wg sync.WaitGroup
    
    for i := 1; i <= 5; i++ {
        wg.Add(1)
        go task(i, &wg)
    }
    
    wg.Wait()  // 等待所有任务完成
    fmt.Println("All tasks completed")
}

// 批量处理示例
func batchProcess(items []int) {
    var wg sync.WaitGroup
    results := make(chan int, len(items))
    
    for _, item := range items {
        wg.Add(1)
        go func(n int) {
            defer wg.Done()
            results <- n * 2  // 简单处理
        }(item)
    }
    
    // 等待所有处理完成
    go func() {
        wg.Wait()
        close(results)
    }()
    
    // 收集结果
    for result := range results {
        fmt.Println("Result:", result)
    }
}
```

- [ ] 用 WaitGroup 实现并发下载 10 个文件的模拟
- [ ] 结合 channel 收集所有 goroutine 的结果
- [ ] 处理 goroutine panic 不影响 WaitGroup 计数的情况

### 5. Context 控制（星期三-星期四）

**核心概念**
- Context 用于传递取消信号、超时、截止时间和请求范围的值
- context.Background() 和 context.TODO()
- context.WithCancel()、WithTimeout()、WithDeadline()
- Context 最佳实践：作为第一个参数传递

**代码示例**
```go
package main

import (
    "context"
    "fmt"
    "time"
)

// 手动取消
func withCancel() {
    ctx, cancel := context.WithCancel(context.Background())
    
    go func() {
        for {
            select {
            case <-ctx.Done():
                fmt.Println("Worker cancelled")
                return
            default:
                fmt.Println("Working...")
                time.Sleep(500 * time.Millisecond)
            }
        }
    }()
    
    time.Sleep(2 * time.Second)
    cancel()  // 取消任务
    time.Sleep(1 * time.Second)
}

// 超时控制
func withTimeout() {
    ctx, cancel := context.WithTimeout(context.Background(), 2*time.Second)
    defer cancel()
    
    select {
    case <-time.After(3 * time.Second):
        fmt.Println("Task completed")
    case <-ctx.Done():
        fmt.Println("Task timeout:", ctx.Err())
    }
}

// 实际应用：数据库查询超时
func queryWithTimeout(ctx context.Context, query string) error {
    // 模拟数据库查询
    done := make(chan struct{})
    
    go func() {
        time.Sleep(3 * time.Second)  // 模拟慢查询
        close(done)
    }()
    
    select {
    case <-done:
        fmt.Println("Query completed:", query)
        return nil
    case <-ctx.Done():
        return fmt.Errorf("query timeout: %w", ctx.Err())
    }
}

func main() {
    ctx, cancel := context.WithTimeout(context.Background(), 2*time.Second)
    defer cancel()
    
    if err := queryWithTimeout(ctx, "SELECT * FROM users"); err != nil {
        fmt.Println("Error:", err)
    }
}
```

- [ ] 改造之前的批量任务，支持超时取消
- [ ] 实现一个可取消的 HTTP 请求
- [ ] 用 context 实现请求链路追踪（存储 trace_id）

### 6. 并发安全（星期四-星期五）

**核心概念**
- 竞态条件（Race Condition）
- 数据竞争（Data Race）
- Mutex 和 RWMutex
- 原子操作 atomic
- 使用 `go run -race` 检测数据竞争

**代码示例**
```go
package main

import (
    "fmt"
    "sync"
    "sync/atomic"
)

// ❌ 错误示例：并发写 map
func unsafeMap() {
    m := make(map[int]int)
    var wg sync.WaitGroup
    
    for i := 0; i < 100; i++ {
        wg.Add(1)
        go func(n int) {
            defer wg.Done()
            m[n] = n * 2  // 并发写，会 panic
        }(i)
    }
    
    wg.Wait()
}

// ✅ 正确示例：使用 Mutex 保护
type SafeCounter struct {
    mu    sync.Mutex
    count int
}

func (c *SafeCounter) Inc() {
    c.mu.Lock()
    defer c.mu.Unlock()
    c.count++
}

func (c *SafeCounter) Value() int {
    c.mu.Lock()
    defer c.mu.Unlock()
    return c.count
}

// ✅ 使用 RWMutex（读多写少场景）
type SafeMap struct {
    mu   sync.RWMutex
    data map[string]int
}

func (m *SafeMap) Set(key string, value int) {
    m.mu.Lock()
    defer m.mu.Unlock()
    m.data[key] = value
}

func (m *SafeMap) Get(key string) (int, bool) {
    m.mu.RLock()  // 读锁
    defer m.mu.RUnlock()
    val, ok := m.data[key]
    return val, ok
}

// ✅ 使用原子操作（简单计数场景）
type AtomicCounter struct {
    count int64
}

func (c *AtomicCounter) Inc() {
    atomic.AddInt64(&c.count, 1)
}

func (c *AtomicCounter) Value() int64 {
    return atomic.LoadInt64(&c.count)
}

func main() {
    // 测试 SafeCounter
    counter := &SafeCounter{}
    var wg sync.WaitGroup
    
    for i := 0; i < 1000; i++ {
        wg.Add(1)
        go func() {
            defer wg.Done()
            counter.Inc()
        }()
    }
    
    wg.Wait()
    fmt.Println("Counter:", counter.Value())
}
```

**实践任务**

- [ ] 用三种方式修复：Mutex、Channel、Atomic

### 任务 1：给小项目添加批量导入功能（星期五）

**需求**
- 支持批量导入 1000 条商品/用户数据
- 使用 goroutine 并发处理，提高导入速度
- 控制并发数为 10，避免数据库连接池耗尽
- 显示导入进度和失败记录

**实现要点**
```go
package service

import (
    "context"
    "fmt"
    "sync"
)

type ImportService struct {
    repo Repository
}

func (s *ImportService) BatchImport(ctx context.Context, items []Item) error {
    const workerCount = 10
    itemCh := make(chan Item, len(items))
    errCh := make(chan error, len(items))
    
    var wg sync.WaitGroup
    
    // 启动 worker 池
    for i := 0; i < workerCount; i++ {
        wg.Add(1)
        go func() {
            defer wg.Done()
            for item := range itemCh {
                select {
                case <-ctx.Done():
                    return
                default:
                    if err := s.repo.Create(item); err != nil {
                        errCh <- fmt.Errorf("import item %s failed: %w", item.ID, err)
                    }
                }
            }
        }()
    }
    
    // 发送任务
    go func() {
        for _, item := range items {
            itemCh <- item
        }
        close(itemCh)
    }()
    
    // 等待完成
    go func() {
        wg.Wait()
        close(errCh)
    }()
    
    // 收集错误
    var errors []error
    for err := range errCh {
        errors = append(errors, err)
    }
    
    if len(errors) > 0 {
        return fmt.Errorf("import failed with %d errors: %v", len(errors), errors[0])
    }
    
    return nil
}
```

- [ ] 导入 1000 条数据耗时在合理范围内（对比串行）
- [ ] 并发数控制在 10 个以内
- [ ] 支持超时取消（5 分钟）
- [ ] 能统计成功/失败数量

### 任务 2：实现异步通知功能（星期五-星期六）

**需求**
- 订单创建后异步发送通知（邮件/短信）
- 通知失败不影响主流程
- 支持重试机制（最多 3 次）
- 记录通知日志

**实现要点**
```go
package service

import (
    "context"
    "log"
    "time"
)

type NotificationService struct {
    queue chan Notification
}

type Notification struct {
    Type    string  // email, sms
    To      string
    Content string
    Retry   int
}

func (s *NotificationService) Start(ctx context.Context) {
    for i := 0; i < 3; i++ {  // 3 个 worker
        go s.worker(ctx)
    }
}

func (s *NotificationService) worker(ctx context.Context) {
    for {
        select {
        case <-ctx.Done():
            return
        case notif := <-s.queue:
            if err := s.send(notif); err != nil {
                log.Printf("Send notification failed: %v", err)
                
                // 重试逻辑
                if notif.Retry < 3 {
                    notif.Retry++
                    time.Sleep(time.Second * time.Duration(notif.Retry))
                    s.queue <- notif
                }
            }
        }
    }
}

func (s *NotificationService) SendAsync(notif Notification) {
    select {
    case s.queue <- notif:
    default:
        log.Println("Notification queue full, dropped")
    }
}
```

**验收标准**
- [ ] 通知发送不阻塞主流程（异步）
- [ ] 支持失败重试
- [ ] Queue 满时有降级策略
- [ ] 可优雅关闭（处理完队列中的任务）

## 五、常见问题与坑

### 1. Goroutine 泄露
```go
// ❌ 忘记关闭 channel，goroutine 永久阻塞
func leak() {
    ch := make(chan int)
    go func() {
        for val := range ch {  // ch 永远不关闭，goroutine 泄露
            fmt.Println(val)
        }
    }()
}
```

### 2. WaitGroup 使用错误
```go
// ❌ Add 在 goroutine 里调用
for i := 0; i < 10; i++ {
    go func() {
        wg.Add(1)  // 错误！可能在 Wait 之后执行
        defer wg.Done()
        // ...
    }()
}
```

### 3. 并发写 map
```go
// ❌ 并发写 map 会 panic
m := make(map[int]int)
go func() { m[1] = 1 }()
go func() { m[2] = 2 }()
```

### 4. Channel 误用
```go
// ❌ 向已关闭的 channel 发送数据会 panic
ch := make(chan int)
close(ch)
ch <- 1  // panic!
```
