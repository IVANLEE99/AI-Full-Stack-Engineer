# 前端半年内转型Go AI全栈工程师 · 周级详细规划

> **核心理念**：你不需要“重新学编程”，而是用Go的语法和思维，重新组织你已经理解的计算逻辑。  
> **前置假设**：公司主力后端语言为 **Golang**，微服务框架可选 **Gin**（入门）或 **go-zero/Kratos**（进阶），本规划以Gin入门，兼顾Go生态通用能力。

---

## 📌 转型心法（先读三遍）
1. **前端优势即壁垒**：你懂SSE/WebSocket（AI流式响应核心）、懂状态机（Agent记忆管理）、懂交互反馈（AI可观测性UI）。这不是包袱，是别人学不来的产品化能力。
2. **AI Agent的本质**：不是炫技调参，而是 **“幂等的工作流 + 可靠的工具调用 + 可审计的权限控制”**。Go的强类型、并发模型和高性能，恰恰是构建生产级Agent的最佳语言。
3. **学会“驾驶AI写代码”**：不要逐行手写Go，善用Cursor/Codex生成骨架，你的核心价值是**代码审查、架构决策和性能调优**。

## 📅 第一阶段：Go语言地基与Web服务（第1-8周）

## ⚠️ 六大避坑指南（Go 特供版）

| 坑点 | 前端思维 | Go 正确姿势 |
| :--- | :--- | :--- |
| **1. 错误处理** | `try/catch` 或 `.catch()` 一把抓。 | **必须显式处理**：`if err != nil { return err }`，绝不忽略错误。 |
| **2. 空指针** | `a.b.c` 报错 `Cannot read properties`。 | Go 没有 `null`，只有 `nil`。访问 `nil` 的 `struct` 字段直接 **Panic**。必须养成 `if obj == nil` 的习惯。 |
| **3. 变量声明** | `let a = 1`。 | `a := 1`（短变量声明）只能在函数内用；包级别必须用 `var`。 |
| **4. 循环引用** | `import` 互相引用，Webpack 能打包。 | Go **禁止**包循环引用，编译直接报错。倒逼你设计更清晰的分层架构。 |
| **5. 并发安全** | `setTimeout` 随便开，单线程不操心。 | 多个 `goroutine` 操作同一 `map` 会 **fatal error: concurrent writes**。必须加锁或改用 `sync.Map`。 |
| **6. 依赖管理** | `npm install` 随便装。 | `go mod tidy` 拉取依赖；注意**版本选型**（v1/v2 不兼容）。 |
