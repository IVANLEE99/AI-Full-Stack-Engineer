## 1. 慢查询治理闭环

```text
发现（慢日志 / APM / 用户投诉）
  → 定性（正确性？偶发？全量？）
  → EXPLAIN / 行数 / 锁等待
  → 改 SQL 或索引或模型
  → 对比证据上线
  → 监控回归
```

开启慢日志（**仅在 lab 或已授权环境**）：

```sql
-- 示意：具体变量以版本为准
SHOW VARIABLES LIKE 'slow_query%';
SHOW VARIABLES LIKE 'long_query_time';
```

优化优先级回顾：

1. 结果正确  
2. 减少扫描行与临时表  
3. 合适索引  
4. 缓存（Redis）  
5. 架构（只读副本、拆分）

## 2. 备份与恢复

| 类型 | 工具直觉 | 用途 |
|------|----------|------|
| 逻辑全量 | `mysqldump` | 小中库、可移植 |
| 物理备份 | XtraBackup 等 | 大库热备 |
| 增量 | binlog | 时间点恢复 PITR |

最小本地演练：

```bash
# 备份
mysqldump -u root -p --single-transaction --routines shop_lab > shop_lab_bak.sql

# 演练恢复到另一库名
mysql -u root -p -e "CREATE DATABASE shop_lab_restore DEFAULT CHARSET utf8mb4"
mysql -u root -p shop_lab_restore < shop_lab_bak.sql
```

**铁律：** 备份成功 ≠ 可恢复；要定期 **恢复演练** 并校验行数/校验和。

生产策略口语版：

```text
每日全量 + 连续 binlog
保留周期满足 RPO/RTO
异地/跨账号一份
恢复剧本写成 runbook
```

## 3. 主从复制（原理级）

```text
主库写 → 记 binlog
  → 从库 IO 线程拉 binlog
  → SQL 线程回放
  → 从库数据追赶主库
```

模式直觉：

- 异步：性能好，主挂可能丢一点未复制事务  
- 半同步：等待至少一个从确认接收  
- 组复制 / InnoDB Cluster：更高阶高可用（了解名词）

**主从延迟：** 从库落后主库若干秒 → 读从库可能读到旧数据。

---

## 4. 读写分离与一致性

```text
写 → 主库
读 → 从库（降压）
```

风险：

```text
用户刚下单 → 立刻读订单列表走从库 → 暂时看不到新订单
```

常见对策：

| 策略 | 说明 |
|------|------|
| 读主 | 写后关键读强制走主 |
| 延迟双读 | 写后 N 秒读主 |
| 会话粘滞 | 同用户短时读主 |
| 可接受最终一致 | 产品允许稍后刷新 |

**结论：** 读写分离是性能手段，不是强一致手段。

## 5. 分库分表决策树

```text
是否真有容量/吞吐问题？
  否 → 优化 SQL/索引/缓存/硬件
  是 → 单表是否过大/热点？
         → 先归档冷数据、分区、只读副本
         → 仍不够再评估分片
```

| 拆分 | 含义 | 例子 |
|------|------|------|
| 垂直 | 按列/业务拆库 | 用户库 / 订单库 |
| 水平 | 同构表按键拆分 | 订单按 user_id 取模 |

分片后的痛点：跨库 JOIN、分布式事务、全局排序分页、二次扩容。  
中间件名词：ShardingSphere、Vitess 等——**7 天内会判断即可，不必会部署**。

全局 ID：号段、雪花、UUID（注意主键无序对 InnoDB 的影响）等。

## 6. 安全迁移（expand / contract）

大表加列示例节奏：

```text
1. 加可空列 / 新表（兼容旧代码）
2. 双写或后台回填（可暂停、可限速）
3. 校验数据
4. 切换读新字段
5. 删旧列或旧路径（contract）
```

Online DDL 仍可能：

- 元数据锁等待
- 磁盘翻倍（拷贝算法时）
- 复制延迟

上线前：备份、窗口、观察指标、回滚开关。

## 8. 与 Redis / 应用分层收束

```text
读：Redis？命中返回 : 读 MySQL → 回填 Redis
写：先成功写 MySQL → 再删除/失效缓存
```

MySQL 是真相源；TTL 只能换「可接受的过期窗口」，不是强一致。

Repository 只表达数据访问；库存是否可下单等决策在 Service。

## 11. 经典面试题速记卡（压缩）

**索引：** B+Tree；最左前缀；覆盖索引；失效（函数、类型转换、前导%）。  
**事务：** ACID；RR 默认；MVCC 快照读；当前读加锁。  
**锁：** 行锁；间隙/临键防幻；死锁回滚+重试。  
**日志：** redo 持久；undo 回滚/MVCC；binlog 复制恢复。  
**架构：** 主从；读写分离一致性；分片最后上。
