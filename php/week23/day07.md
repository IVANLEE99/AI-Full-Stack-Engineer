### 1.4 为什么需要 Docker（预习）

先理解 Docker 要解决的痛点：

```text
没有 Docker 的世界：
  你机器：PHP 8.3 + 装了某扩展 → 能跑
  同事机器：PHP 8.1 + 没装扩展 → 报错
  服务器：PHP 7.4 → 直接崩

  经典甩锅现场：「在我这明明能跑啊！」

有了 Docker：
  把「代码 + PHP 8.3 + 扩展 + 依赖」一起打包成一个镜像
  任何机器只要能跑 Docker，行为完全一致
```

小白重点：

> Docker 解决的是「环境不一致」这个老大难。它把运行环境和代码封装在一起，做到「一次构建，处处运行」。这对交付平台特别重要，因为别人拿到你的镜像就能直接跑，不用配环境。

---

### 1.5 镜像、容器、Dockerfile、Compose 的关系（预习）

先建立四个核心概念的关系：

| 概念 | 是什么 | 类比 |
|---|---|---|
| Dockerfile | 构建镜像的「食谱」 | 菜谱 |
| 镜像 Image | 打包好的只读模板 | 做好的预制菜 |
| 容器 Container | 镜像运行起来的实例 | 端上桌加热的那盘菜 |
| docker-compose | 一次编排多个容器 | 一整桌宴席的安排 |

```text
关系链：
  写 Dockerfile → 构建出 Image → 运行 Image 得到 Container
  多个 Container（PHP + 数据库 + 缓存）→ 用 docker-compose 一起管
```

Node.js 类比：

```text
Dockerfile      ≈ 描述怎么装 Node 环境的脚本
Image           ≈ 打包好的 node_modules + 代码 + 运行时
Container       ≈ node app.js 跑起来的那个进程
docker-compose  ≈ 同时起 app + mongo + redis 的编排
```

小白重点：

> 记住这条链：Dockerfile（食谱）→ Image（预制菜）→ Container（端上桌的菜）。Compose 是管理一整桌菜（多个服务）的工具。明天正式学时不会懵。

### 1.6 预习：PHP 平台的 Dockerfile

先看一眼将来要写的 Dockerfile 大概长什么样（现在只求看懂，不求会写）：

```dockerfile
# 基础镜像：官方 PHP 8.3 + CLI
FROM php:8.3-cli

# 装 Composer（依赖管理工具）
COPY --from=composer:latest /usr/bin/composer /usr/bin/composer

# 设置工作目录
WORKDIR /app

# 先复制依赖清单并安装（利用缓存，代码变了不重装依赖）
COPY composer.json composer.lock ./
RUN composer install --no-dev --optimize-autoloader

# 再复制项目代码
COPY . .

# 容器启动时默认执行的命令
CMD ["php", "index.php", "订单 SO001 到哪了"]
```

逐行读懂：

| 指令 | 作用 |
|---|---|
| `FROM` | 选一个现成的基础环境（自带 PHP 8.3） |
| `COPY --from` | 从别的镜像拷贝 Composer 过来 |
| `WORKDIR` | 设定容器内的工作目录 |
| `COPY composer.*` + `RUN composer install` | 先装依赖（分层缓存优化） |
| `COPY . .` | 把项目代码拷进镜像 |
| `CMD` | 容器启动默认跑什么 |

小白重点：

> 「先复制 composer.json 装依赖，再复制代码」是一个重要技巧：Docker 是分层缓存的，依赖没变就不重装，只有代码变时重建代码层，构建快很多。这和 Node 里「先 COPY package.json 再 npm install」是同一个套路。

### 1.7 预习：docker-compose.yml

如果平台以后需要数据库、缓存，就用 Compose 一次性编排：

```yaml
services:
  # 我们的 Agent 平台
  agent:
    build: .                 # 用当前目录的 Dockerfile 构建
    volumes:
      - .:/app               # 把本地代码挂进容器（改代码即时生效）
    depends_on:
      - db

  # 一个数据库服务（示例）
  db:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: example
      MYSQL_DATABASE: agent
    ports:
      - "3306:3306"
```

启动整套（明天会实操，今天只看）：

```bash
docker compose up
```

小白重点：

> Compose 让你用一个 yml 描述「我要哪些服务、它们怎么连」，然后一条 `docker compose up` 全起来。这比手动一个个 `docker run` 省心太多。明天正式学。

## 4. JS/Node.js 类比

| PHP / Docker | Node.js 类比 | 说明 |
|---|---|---|
| 验收清单 | 上线前 checklist | 客观逐项确认 |
| Dockerfile | 描述 Node 环境的构建脚本 | 环境的食谱 |
| `php:8.3-cli` 基础镜像 | `node:20` 基础镜像 | 现成运行时 |
| `composer install` in Dockerfile | `npm install` in Dockerfile | 装依赖层 |
| 先 COPY composer.json | 先 COPY package.json | 分层缓存技巧 |
| docker-compose | 同时起 app + db + redis | 多服务编排 |
| `docker compose up` | `docker compose up` | 一键全起 |
