### 1.6 预习 BFF：为什么下一阶段要学网关？

前 4 周主要关注单体/基础项目阅读。

下一阶段会进入：

```text
BFF 网关 + 微服务
```

你要先理解一个问题：如果前端页面需要多个后端服务的数据，应该怎么办？

可能的结构：

```text
前端
  ↓
BFF / 网关
  ↓
用户服务 / 商品服务 / 订单服务 / 支付服务
```

BFF 会负责：

- 接收前端请求
- 做登录校验
- 聚合多个服务数据
- 统一响应格式
- 适配前端页面需要的数据结构

但核心业务规则仍然应该放在具体服务里。

## 4. JS/Node.js 类比

- 阶段复盘 ≈ 里程碑回顾
- CSR 链路 ≈ Express Controller → Service → Repository → ORM
- Composer autoload ≈ npm package resolution + import
- `g_config()` ≈ config service / feature flags
- BFF ≈ 专门服务前端的 Express/NestJS API 聚合层
