## 七、附录 A：仓库代号与分层规范

### 仓库代号表

| 代号 | 技术栈 | 说明 |
|------|--------|------|
| mall-gateway | Yii2 | C 端/运营 API 网关（BFF） |
| mall-core | Yii2 Advanced | 核心微服务集群（order/goods/market/site 等子模块） |
| pay-service | Yii2 Advanced | 独立支付微服务 |
| user-service | Yii2 Advanced | 独立用户微服务 |
| aftersale-service | Yii2 Advanced | 独立售后微服务 |
| store-api | ThinkPHP 8 | 门店管理后端 API |
| ai-workspace | 文档/配置 | AI 开发工作空间 |
| mcp-server | TypeScript | MCP Server 集合 |
| skill-library | Markdown | Agent 通用技能库 |

### 统一分层（Yii2 系）

```
Controller（编排 + VO 格式化）
  → Service（业务逻辑）
    → Repository（DB 操作）
      → Model / ActiveRecord（表映射）
```

- 跨服务：`common/api/*Internal.php`（禁止 Service 直接 HTTP）
- 配置：`g_config(ConfigHelper::$MODULE, key, default)`
- 日志：`g_log_info()` / `g_log_warning()` / `g_log_error()`
- 响应：`endSuccess(data)` / `endFail(code, msg)`

### ThinkPHP 8 分层（store-api）

```
Controller → Validate → Service → Model
```

- 响应：`g_ok_data()` / `g_err_data()`
- 跨服务：`InternalServiceHelper`

## 八、附录 B：PHP ↔ JS/Node 概念对照总表

| # | PHP / Yii2 概念 | JS/Node 类比 | 差异要点 |
|---|----------------|-------------|----------|
| 1 | `composer.json` | `package.json` | Composer 用 PSR-4 映射 namespace → 目录 |
| 2 | `composer install` | `npm install` | Composer 有 `vendor/`，npm 有 `node_modules/` |
| 3 | `namespace App\Services` | ES Module 作用域 | PHP namespace 是语言级，JS 靠文件路径 |
| 4 | `use App\Services\UserService` | `import UserService from '...'` | PHP use 是编译期别名 |
| 5 | `strict_types=1` | TypeScript `strict: true` | PHP 运行时类型检查，TS 编译期 |
| 6 | `class UserService {}` | `class UserService {}` | 语法几乎一致 |
| 7 | `interface` | TypeScript `interface` | PHP interface 可被 class implements |
| 8 | `trait Loggable` | Mixin / composable 抽逻辑 | PHP trait 是语言特性，JS 靠组合 |
| 9 | `enum Status: int` | TS `enum` / union type | PHP 8.1+ enum 可有方法 |
| 10 | `readonly class` | `Object.freeze()` / TS `readonly` | PHP 8.2+ 类级别不可变 |
| 11 | `match` 表达式 | `switch` + 立即返回 | match 严格比较且必须穷尽 |
| 12 | `?->` 空安全操作符 | `?.` 可选链 | 语法相似，PHP 8.0+ |
| 13 | `Controller::actionXxx` | `router.get('/xxx', handler)` | Yii2 约定 action 前缀 |
| 14 | `behaviors()` | Express `app.use(middleware)` | behaviors 可配多个 Filter |
| 15 | `beforeAction` 钩子 | middleware `(req,res,next)` | 返回值可中断请求 |
| 16 | `BaseForm` 校验 | Zod `.parse()` / Joi `.validate()` | Form 有 rules() + scenarios |
| 17 | `Yii::$app->request` | `req` (Express) | 封装了 GET/POST/JSON body |
| 18 | `Service::instance()` | NestJS `@Injectable()` 单例 | Yii2 通过 DI 容器管理 |
| 19 | `Repository` | Prisma Repository 模式 | Repository 只做 DB，不做业务 |
| 20 | `ActiveRecord` | Sequelize `Model` | AR 把 DB 操作绑在 Model 类上 |
| 21 | `->where()->one()` | `Model.findOne({ where })` | 链式查询 API 相似 |
| 22 | `g_config(m, k, d)` | `process.env.KEY ?? default` | g_config 读配置中心，支持热更新 |
| 23 | `g_log_info(f, m, ctx)` | `logger.info(m, ctx)` | 按文件名分日志 |
| 24 | `endSuccess(data)` | `res.json({ code:0, data })` | 统一响应格式约定 |
| 25 | `*Internal.php` | `axios.create({ baseURL })` | 微服务间 HTTP 封装 |
| 26 | `RabbitMq::send()` | `channel.publish()` (amqplib) | 异步消息投递 |
| 27 | `MqConsumeController` | queue consumer webhook | HTTP 被 MQ 消费者回调 |
| 28 | `NodeExecutionEngine` | middleware pipeline | 按序执行 Node，可中断 |
| 29 | `processingType` 策略 | Strategy Pattern classes | 替代大型 switch-case |
| 30 | `LockHandleRedis` | Redlock / SET NX EX | 分布式锁防重复提交 |
| 31 | `yii controller/action` | `node scripts/xxx.js` | CLI 批处理 |
| 32 | `PaymentFactory` | Factory Pattern / 策略注册表 | 按支付公司实例化 SDK |
| 33 | ThinkPHP `Validate` | `class-validator` decorators | scene 分组校验 |
| 34 | FastAPI | Express / Fastify | Python async HTTP 框架 |
| 35 | MCP Tool | OpenAI `tools[]` / function calling | 给 LLM 注册可调用函数 |
| 36 | RAG Embedding | 向量 DB + `similaritySearch` | 语义搜索替代关键词 |
| 37 | LangGraph State | Redux store / 状态机 | 节点间共享状态对象 |
| 38 | Laravel Middleware | Express middleware | 框架不同，模式相同 |
| 39 | Laravel Eloquent | Sequelize / Prisma | ORM 抽象层 |
| 40 | Laravel Queue | BullMQ / Bull | 异步任务队列 |
