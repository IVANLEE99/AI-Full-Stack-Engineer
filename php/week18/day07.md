### 1.3 终验：准确率目标

跑一遍完整评估，确认最终准确率：

```python
from faq_agent import FaqAgent
from eval_set import EVAL_SET
from knowledge_base import DOCS

agent = FaqAgent(DOCS)
hit = 0
for item in EVAL_SET:
    result = agent.answer(item["question"])
    got_ids = {s["doc_id"] for s in result["sources"]}
    if item["expected_doc_id"] in got_ids:
        hit += 1

print(f"最终准确率：{hit}/{len(EVAL_SET)} = {hit/len(EVAL_SET):.0%}")
```

达标标准：准确率 > 70%（本周基线目标）。

小白重点：终验用的是完整链路（Hybrid + Rerank），如果没达标，回看 Day05 的失败 case 分析，通常问题出在召回或权重上。

### 1.4 本周三个核心认知

复盘时，用自己的话说清这三点：

1. **为什么要 Hybrid**：语义检索擅长“意思相近”，但对专有名词、编号、缩写不敏感；BM25 擅长精确关键词。两路融合互补。
2. **Rerank 的作用**：粗召回（向量/BM25）追求“快而全”，可能把答非所问的干扰项排前面；Rerank 用更重的 cross-encoder 把真正相关的顶上来。
3. **引用溯源治幻觉**：让回答标注 [1][2] 对应源文档，用户能核对，LLM 也更不敢瞎编。

小白重点：能用大白话讲清这三点，本周就算真正学会了。

---

### 1.5 预习 Memory：多轮对话为什么需要记忆

本周的 FAQ Agent 是“一问一答”，每次都从零检索。但真实对话是这样的：

```text
用户：我想退货
助手：请问是哪个订单？
用户：上周买的那双鞋       ← 这里“那双鞋”指代前文，需要记住上下文
```

如果没有记忆，助手会忘掉“退货”这个话题，答非所问。

小白重点：Memory 要解决的是“**跨轮次记住上下文**”，让 Agent 像人一样能接住“它、那个、上一个”这类指代。

---

### 1.6 检索记忆 vs 对话记忆

别把两种“记忆”搞混：

| 类型 | 存什么 | 解决什么 | 本周/下周 |
|---|---|---|---|
| 检索记忆（RAG） | 知识库文档 | 让模型答出它没训练过的知识 | 本周（已学） |
| 对话记忆（Memory） | 历史对话轮次 | 让模型记住这次对话说过啥 | 下周（预习） |

对话记忆常见做法：

```python
# 把历史对话拼进 prompt（最朴素的 Memory）
messages = [
    {"role": "user", "content": "我想退货"},
    {"role": "assistant", "content": "请问是哪个订单？"},
    {"role": "user", "content": "上周买的那双鞋"},  # 模型能看到前两轮
]
```

小白重点：最简单的 Memory 就是“把历史对话也塞进 prompt”。下周会学更省 token 的做法（摘要、窗口、向量化历史）。

## 4. JS/Node.js 类比

| 概念 | Node.js 类比 | 说明 |
|---|---|---|
| 本周验收 | 项目上线前的 checklist | 逐条勾，不靠感觉 |
| 检索记忆（RAG） | 查数据库/外部知识 | 拿模型没有的知识 |
| 对话记忆（Memory） | session / 会话状态 | 记住这次交互的上下文 |
| 把历史拼进 prompt | 请求里带 session 数据 | 让下一步知道之前发生了啥 |
| prompt 超长 | session 数据膨胀 | 需要裁剪/摘要策略 |

小白重点：对话记忆之于 Agent，就像 session 之于 Web 服务——都是为了“记住上一步发生了什么”，否则每个请求都是陌生人。

## 8. 今日自测题

### 8.1 本周这条检索链路每一环解决什么问题？

参考答案：

> ✅ BM25 补语义检索对专有名词/编号不敏感的短板；Hybrid 融合两路互补；Rerank 修粗召回排序不准；引用溯源治 LLM 幻觉、让答案可核对。

---

### 8.2 检索记忆和对话记忆有什么区别？

参考答案：

> ✅ 检索记忆（RAG）存知识库文档，解决“答出没训练过的知识”；对话记忆（Memory）存历史对话轮次，解决“记住这次对话说过啥”。

---

### 8.3 最朴素的 Memory 怎么实现？

参考答案：

> ✅ 把历史对话轮次一起拼进 prompt（messages 列表），让模型能看到前几轮说了什么。缺点是对话越长 prompt 越长。

---

### 8.4 终验准确率没达标，先查哪里？

参考答案：

> ✅ 回看 Day05 的失败 case 分析。通常问题出在召回（相关文档没被召回）或 Hybrid 权重上，其次才是生成环节。

---

### 8.5 为什么要带着问题预习下一周？

参考答案：

> ✅ 带着具体问题学，能在听到答案时立刻对上，比从零被动听效率高得多，也更容易发现知识盲点。
