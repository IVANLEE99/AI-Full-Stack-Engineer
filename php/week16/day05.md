### 1.2 编排对比文档：核心对比表

这是今天要产出的文档主体。逐个维度对比。

| 维度 | 图编排（LangGraph） | 链式编排（NodeExecutionEngine） |
|------|--------------------|------------------------------|
| 结构 | 有向图（节点 + 边） | 线性链（Node 数组） |
| 共享状态 | State（dict/TypedDict） | Context（PHP 对象） |
| 单个步骤 | Node（函数） | Node（类的 handle 方法） |
| 决定下一步 | Edge / 条件边 | 数组顺序 + 节点内 if |
| 分支 | 一等公民（条件边） | 写在节点内部的 if |
| 循环 | 天然支持（边可指回去） | 不擅长（要靠外层 while） |
| 可视化 | 能导出成图 | 靠人工画 |
| 典型语言 | Python | PHP |
| 典型场景 | AI Agent、多轮决策 | 支付、售后等固定业务流 |

---

### 1.3 三处「本质相同」

不管画法多不同，两者骨子里是一回事：

1. **共享状态贯穿全程**：图有 State，链有 Context，都是「一份数据从头传到尾」。
2. **步骤单一职责**：图的 Node 和链的 Node，都强调一个节点只干一件事。
3. **流程可组合**：都是「把大流程拆成小步骤，再组装起来」。

```text
        State/Context（共享状态）
              │
   ┌──────────┼──────────┐
 步骤A ──▶ 步骤B ──▶ 步骤C     ← 本质都是这个
```

---

### 1.4 三处「关键不同」

1. **分支的地位不同**
   - 图：分支是「一等公民」，用条件边显式表达，一眼看得出有几条路。
   - 链：分支「藏在节点里」，用 if 表达，得读代码才知道。

2. **循环能力不同**
   - 图：边可以指回之前的节点，天然支持「反复执行直到满足条件」（AI Agent 的反思循环靠这个）。
   - 链：单向往前，想循环得在链外面套 while，很别扭。

3. **可视化不同**
   - 图：结构本身就是图，能直接画/导出。
   - 链：得靠人脑或人工画，代码本身不「自带图」。

---

### 1.5 选型决策树：什么时候用哪种

这是文档最有价值的部分——给出可操作的判断。

```text
你的流程……
│
├── 是一条基本笔直的线，最多几个 if 分支？
│      └──▶ 用【链式编排】（PHP Node 链够了，简单直接）
│
├── 有明显的「多路分支」，需要一眼看清所有路径？
│      └──▶ 倾向【图编排】（条件边更清晰）
│
├── 需要「循环/重试直到满足条件」（比如 Agent 反复思考）？
│      └──▶ 用【图编排】（链做不了漂亮的循环）
│
└── 需要把流程画给产品/非技术看，或需要可观测？
       └──▶ 倾向【图编排】（自带可视化）
```

一句话决策：

> 固定业务流（支付、售后）用链；AI Agent 这种「走一步看一步、可能要回头」的流程用图。

---

### 1.6 常见误区纠正

| 误区 | 纠正 |
|------|------|
| 「图编排比链式先进，应该都用图」 | ❌ 简单流程用图是过度设计，链式更省事 |
| 「PHP 不能做图编排」 | ❌ 能，只是社区里 Python + LangGraph 更成熟 |
| 「LangGraph 只能做 AI」 | ❌ 它是通用编排框架，只是 AI 场景最常用 |
| 「链式不能分支」 | ❌ 能，用节点内 if，只是不如条件边直观 |
| 「State 和 Context 是两种东西」 | ❌ 本质一样，都是「贯穿全程的共享状态」 |

## 4. JS/Node.js 类比

本周类比总收口：

| 概念 | PHP | LangGraph | JS/Node 类比 |
|------|-----|-----------|-------------|
| 共享状态 | Context 对象 | State dict | Redux store / Express req |
| 单个步骤 | Node 类 | Node 函数 | middleware 函数 |
| 步骤连接 | Node 数组顺序 | Edge | `app.use()` 顺序 |
| 分支 | 节点内 if | 条件边 | if / 路由 |
| 循环 | 外层 while | 边指回去 | while / 递归调用 |

一句话：

> 三套东西（PHP Node 链、LangGraph 图、Express 中间件）都是同一个思想的不同长相：**共享状态 + 有序步骤 + 分支控制**。

## 8. 今日自测题

### 8.1 图编排和链式编排，哪个更「先进」？

参考答案：

> ✅ 没有谁更先进。它们面对不同复杂度：简单固定流程用链更省事，有循环/多分支/动态决策（如 AI Agent）用图更合适。选型看场景，不看新旧。

---

### 8.2 链式编排为什么不擅长「循环」？

参考答案：

> ✅ 链是单向往前的 Node 数组，一个接一个执行，没有「指回之前节点」的机制。想循环只能在链外面套 while，很别扭。图编排的边可以指回去，所以天然支持循环。

---

### 8.3 State 和 Context 是两种不同的东西吗？

参考答案：

> ✅ 不是。本质完全一样——都是「贯穿整个流程、被每个步骤读写的共享状态」。只是 LangGraph 里叫 State，PHP Node 链里叫 Context。

---

### 8.4 什么场景明确应该选图编排？

参考答案：

> ✅ 需要「循环/反复执行直到满足条件」（典型如 AI Agent 反思重试）、需要「多路分支且要一眼看清所有路径」、或需要「把流程可视化给非技术看」。

---

### 8.5 「PHP 不能做图编排」这句话对吗？

参考答案：

> ✅ 不对。PHP 完全能做图编排，只是社区生态里 Python + LangGraph 更成熟。PHP 业务流大多是固定流程，用链式就够了，所以图编排不常见。
