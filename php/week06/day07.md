### 1.3 订单域必须掌握的 5 个风险

| 风险 | 你要能解释什么 |
|---|---|
| 金额 | 前端金额不能信，提交订单时后端必须重新计算 |
| 幂等 | 重复请求不能创建重复订单 |
| 库存 | 下单时要校验/扣减/锁定库存，防止超卖 |
| 状态流转 | 订单状态只能按合法路径变化 |
| 支付关系 | 订单创建后通常进入待支付，支付成功再改变状态 |

如果这 5 个说不清，说明订单域还没真正掌握。

### 1.4 口述下单主链路

请尝试不看笔记口述：

```text
用户进入结账页，前端请求 confirm。
网关注入用户和站点信息后转发到订单服务。
OrderController 接收请求，Form 校验基础参数。
OrderService 查询商品、地址、优惠、库存，计算确认页金额并返回。
用户点击提交订单，前端请求 place。
后端再次校验参数和业务数据，获取锁或幂等 token。
OrderService 创建订单主表和明细，初始化待支付状态。
返回 order_id 或支付准备信息，后续进入支付域。
```

你能口述出来，说明本周主线已经建立。

### 1.5 预习支付域

下周将进入支付域。支付域和订单域强相关。

你需要提前知道：

| 支付问题 | 为什么重要 |
|---|---|
| 支付单和订单是什么关系？ | 一笔订单可能有支付记录 |
| 支付状态如何回写订单？ | 支付成功后订单状态变化 |
| 支付回调如何验签？ | 防止伪造回调 |
| 重复回调如何处理？ | 支付平台可能多次通知 |
| 金额如何校验？ | 防止金额不一致 |

预习对象：

- `PayService`
- 支付 Controller
- 支付回调接口
- 支付状态常量
- 订单与支付关联字段
