## 1. 学习内容

### 1.1 本周主线复盘：为什么需要 Memory

从根上说一遍：

```text
LLM 本身是无状态的（stateless）：
每次请求都是独立的，它不记得上一次说过什么。

所以「记忆」不在模型里，在你的后端：
你要把历史对话，每次都拼进请求里发给模型。
```

一句话：

> 记忆是「后端工程」问题，不是「模型能力」问题。你负责存历史、挑历史、压历史。

---

### 1.2 三层能力串讲

本周做的三件事，串成一条链：

| 层 | 做什么 | 解决的问题 | 对应 Day |
|---|---|---|---|
| 存 | SessionManager 存全量历史 | 会话隔离、能找回 | Day01 |
| 截 | 上下文窗口只带最近 N 条 | token 超限、成本 | Day02 |
| 压 | 超长时 LLM 生成摘要 | 保留旧信息又不爆窗口 | Day03 |
| 验 | 5 轮连贯对话测试 | 证明记忆真的生效 | Day04/06 |

完整链路：

```python
def chat(user_input):
    session.append("user", user_input)     # 存
    maybe_summarize(session)               # 压
    context = build_window(session, N)     # 截
    reply = call_llm(context)              # 用
    session.append("assistant", reply)     # 存
    return reply
```

### 1.6 预习：Multi-Agent 是什么

下周主题：多智能体协作。

先建立直觉：

```text
单 Agent：一个 AI 干所有事（既查订单、又答政策、又算价格）。
Multi-Agent：多个专职 AI 分工协作。

比如客服场景：
- 分诊 Agent：判断用户意图，决定交给谁
- 订单 Agent：专门查订单
- 政策 Agent：专门答退换货政策
- 汇总 Agent：整合结果回给用户
```

一句话：

> Multi-Agent ≈ 把一个「全能但容易糊涂」的 AI，拆成一个「团队」，各管一摊，再协调。

---

### 1.7 预习：Memory 在多 Agent 里的角色

本周的 Memory 下周还要用，但更复杂：

| 问题 | 单 Agent | Multi-Agent |
|---|---|---|
| 谁持有记忆 | 一个 Session | 共享 Session？各自 Session？ |
| 上下文怎么传 | 直接带全部 | 只给每个 Agent 它需要的部分 |
| 摘要给谁 | 给自己 | 可能要给协调者 |

思考题（先想，不用答完美）：

```text
如果订单 Agent 和政策 Agent 是两个独立的 AI，
它们要不要共享同一份对话历史？
如果共享，会不会互相干扰？如果不共享，用户说过的话怎么传递？
```

小白重点：

> 带着「记忆怎么在多个 Agent 间流动」这个问题进入下周，你会学得更快。

## 4. JS/Node.js 类比

| Python / AI 后端 | Node.js 类比 | 说明 |
|---|---|---|
| LLM 无状态 | HTTP 无状态 | 靠外部存储维持状态 |
| SessionManager | express-session | 按 id 存取会话 |
| 上下文窗口 | 分页 / 滑动窗口 | 只处理一段数据 |
| 摘要压缩 | 日志归档压缩 | 旧数据压缩存档 |
| Multi-Agent | 微服务 / 中间件链 | 拆成多个专职单元协作 |
| 分诊 Agent | API Gateway 路由 | 按意图分发请求 |

一句话类比：

> 从单 Agent 到 Multi-Agent，就像从「一个大单体应用」拆成「多个微服务」——各司其职，但要解决服务间怎么共享状态（记忆）的问题。

## 8. 今日自测题

### 8.1 一句话概括本周 Memory 系统？

参考答案：

> ✅ 存全量历史（SessionManager）、只带最近 N 条控 token（窗口）、超长时压成摘要（摘要压缩），让无状态的 LLM 表现得有记忆。

---

### 8.2 为什么记忆是后端问题不是模型问题？

参考答案：

> ✅ 因为 LLM 每次请求独立、不记上一次。记忆靠后端每次把历史拼进请求，所以存/挑/压历史都是后端工程。

---

### 8.3 存、截、压分别解决什么问题？

参考答案：

> ✅ 存解决会话隔离和找回；截解决 token 超限和成本；压解决旧信息保留又不爆窗口。

---

### 8.4 Multi-Agent 和单 Agent 的核心区别？

参考答案：

> ✅ 单 Agent 一个 AI 干所有事；Multi-Agent 是多个专职 AI 分工协作，再由协调机制整合，类似从单体拆成微服务。

---

### 8.5 进入多 Agent 后，Memory 会多出什么新问题？

参考答案：

> ✅ 谁持有记忆、多个 Agent 是否共享历史、如何只把各 Agent 需要的上下文传给它、摘要给谁——即记忆在多个 Agent 间如何流动与隔离。
