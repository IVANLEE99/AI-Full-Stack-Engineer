### 1.5 什么是 PRD

PRD = Product Requirement Document，产品需求文档。它是"把一个产品想法写清楚"的标准文档。

一份基础 PRD 通常包含：

```text
1. 背景与目标   —— 为什么要做，解决什么问题
2. 目标用户     —— 给谁用
3. 功能清单     —— 要做哪些功能（核心/次要）
4. 用户流程     —— 用户怎么一步步用
5. 非功能需求   —— 性能、安全、兼容性等
6. 里程碑       —— 分几期做，每期做什么
```

小白重点：你会发现，PRD 的"功能清单"和本周需求 Agent 输出的 `features` 几乎是一回事。需求 Agent 其实就是"自动写 PRD 草稿的机器"。

---

### 1.6 PRD 模板（预习）

下面是一个可以直接套用的最小 PRD 模板：

```markdown
# 项目名称：XXX

## 一、背景与目标
- 要解决的问题：
- 期望达成的目标：

## 二、目标用户
- 主要用户：
- 使用场景：

## 三、功能清单
### 核心功能（MVP）
- [ ] 功能 1
- [ ] 功能 2

### 次要功能（后续）
- [ ] 功能 3

## 四、用户主流程
1. 用户打开 → 
2. 用户操作 → 
3. 系统响应 → 

## 五、非功能需求
- 性能：
- 安全：
- 兼容性：

## 六、里程碑
- 第一期（MVP）：
- 第二期：
```

小白重点：毕业项目就是要你独立写出这样一份 PRD，然后照着它做。本周的 Multi-Agent 流水线，可以帮你自动生成 PRD 的初稿。

---

### 1.7 需求 Agent 输出 → PRD 的映射

把本周需求 Agent 的输出，对照 PRD 的字段：

| 需求 Agent 输出字段 | PRD 对应章节 |
|---|---|
| `project_name` | 项目名称 |
| `background` | 背景与目标 |
| `target_users` | 目标用户 |
| `features` | 功能清单 |
| `user_flow` | 用户主流程 |
| `non_functional` | 非功能需求 |

代码上，你甚至可以写一个"需求 Agent 输出 → PRD Markdown"的转换器：

```python
def to_prd_markdown(req: dict) -> str:
    lines = [f"# 项目名称：{req.get('project_name', 'XXX')}", ""]
    lines.append("## 三、功能清单")
    for f in req.get("features", []):
        lines.append(f"- [ ] {f}")
    return "\n".join(lines)
```

小白重点：这就是本周和毕业项目的桥梁——需求 Agent 不只是练习，它是你写 PRD 的生产力工具。

## 4. JS/Node.js 类比

| 本周概念 | Node.js 类比 | 说明 |
|---|---|---|
| 验收清单 | 测试用例清单 / CI 检查项 | 用证据确认功能达标 |
| `>` 重定向存输出 | `fs.writeFileSync` | 把结果落盘 |
| PRD | 项目 README / RFC 文档 | 把想法写清楚 |
| 需求 Agent → PRD | 脚手架生成初始文档 | 自动产出初稿 |
| 遗留问题清单 | issue backlog | 待办与技术债 |

小白重点：验收和写文档，看起来"不是写代码"，但恰恰是资深工程师和新手的分水岭。会写清楚、能验收、留得下证据，才是能独立交付的人。

## 8. 今日自测题

### 8.1 什么叫"用证据验收"？

参考答案：

> ✅ 每一条完成标准，都能指着一个具体的文件或一段可复现的输出说"这就是证据"，而不是凭感觉说"我觉得做完了"。这是工程师用证据说话的基本习惯。

---

### 8.2 本周 Multi-Agent 的核心思想是什么？

参考答案：

> ✅ 把一个大任务拆成多个角色（需求、架构、Review），让它们像流水线一样接力协作，每个 Agent 只专注自己那一段。

---

### 8.3 PRD 通常包含哪些部分？

参考答案：

> ✅ 背景与目标、目标用户、功能清单、用户流程、非功能需求、里程碑。核心是把"要做什么、给谁做、怎么做"写清楚。

> ✅ 需求 Agent 的输出字段（project_name、features、target_users 等）几乎可以直接映射到 PRD 的章节。需求 Agent 本质上就是"自动写 PRD 初稿的机器"。

### 8.5 为什么"会写文档、能验收"是工程师的分水岭？
