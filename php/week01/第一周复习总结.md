## 1. 5 分钟速览

### 1.1 一周最重要的 12 句话

1. PHP 变量以 `$` 开头，字符串拼接使用 `.`，常见类型有 `string`、`int`、`float`、`bool`、`array` 和 `null`。
2. `declare(strict_types=1);` 配合参数类型和返回类型，能减少隐式类型转换造成的错误。
3. Composer 既是依赖管理工具，也负责生成自动加载器：`composer.json` 类似 `package.json`，`vendor/` 类似 `node_modules/`。
4. `require __DIR__ . '/vendor/autoload.php';` 引入 Composer 自动加载入口。
5. `namespace` 给类分组并避免同名冲突；`use` 给完整类名建立短别名，本身不负责加载文件。
6. PSR-4 规定命名空间前缀到目录的映射，例如 `App\` → `src/`。
7. OOP 用类封装数据和行为；继承表达“是一种”，接口表达“具备某种契约”，抽象类表达“可复用的半成品基类”。
8. `$this->` 访问对象成员，`::` 访问类级成员；`self` 指当前定义类，`static` 可体现后期静态绑定。
9. Trait 用于横向复用能力；Exception 用于表达无法正常继续的异常流程。
10. Controller 接请求，Service 处理业务，Repository 访问数据，Model/ORM 映射或操作数据库。
11. PSR-12 统一代码排版；PSR-4 统一自动加载映射，两者不是一回事。
12. REST API 用 HTTP 方法操作资源；Todo 实战覆盖列表、详情、创建、更新、删除和统一 JSON 响应。

### 1.2 必记命令

```bash
php -v
composer -V
composer install
composer dump-autoload
php example.php
php -S localhost:8000 -t public
curl http://localhost:8000/todos
```

### 1.3 必记工程链路

```text
HTTP 请求
  → public/index.php（入口与简易路由）
  → Controller（接收请求、提取参数）
  → Service（业务逻辑）
  → Repository（数据访问；Todo 练习尚未接入）
  → Model / ORM / Database
  → Response（状态码 + 统一 JSON）
```

```text
完整类名 App\Services\UserService
  → 匹配 composer.json 中 App\ => src/
  → 剩余 Services\UserService
  → src/Services/UserService.php
```

### 1.4 本周能力边界

本周已经建立 PHP 工程心智模型，但 Todo API 仍是练习版：

- 数据保存在请求内的数组中，没有持久化；
- 路由、参数校验和异常处理较简化；
- 尚未系统学习测试框架、数据库迁移、认证、队列和框架级中间件；
- Day 7 的 Yii2 生命周期只是预习，不应当视为已经掌握。

## 3. PHP 基础、类型与调试（Day 1）

### 3.1 文件与变量

纯 PHP 文件通常这样开头，并省略结束标签 `?>`：

```php
<?php

declare(strict_types=1);

$name = 'Tom';
$age = 18;

echo $name . ' is ' . $age . PHP_EOL;
```

省略 `?>` 可以避免文件尾部空格或换行造成意外输出，尤其能减少 `headers already sent` 和 JSON 响应污染问题。

### 3.2 常见类型

```php
$name = 'Alice';       // string
$age = 20;             // int
$price = 19.99;        // float
$isVip = true;         // bool
$items = [1, 2, 3];    // array
$user = null;          // null
```

PHP 的 `array` 同时承担两类用途：

```php
$names = ['Tom', 'Jerry'];          // 类似 JS Array
$user = ['name' => 'Tom', 'age' => 18]; // 类似 JS Object
```

### 3.3 参数类型、返回类型与 `strict_types`

```php
<?php

declare(strict_types=1);

function add(int $a, int $b): int
{
    return $a + $b;
}

echo add(1, 2);
```

关键理解：

- `int $a` 是参数类型；
- `: int` 是返回类型；
- PHP 的类型约束在运行时生效；
- 严格模式能减少标量参数被自动转换的情况，例如把 `"1"` 当作 `1`；
- 严格模式不能替代输入校验，HTTP 请求中的原始数据仍要主动验证和转换。

现代 PHP 的质量方向是：能写明确类型时就写明确类型，避免无理由省略参数、属性和返回类型。

### 3.4 `mixed` 的使用边界

`mixed` 表示该位置确实允许任意类型，并且已经包含 `null`：

```php
function jsonResponse(mixed $data = null): void
{
    // ...
}
```

适合统一响应、序列化等通用入口；业务参数如果能确定为 `string`、`int`、`array` 等，就不要滥用 `mixed`。

### 3.5 PHP 8 特性速记

#### `match`

`match` 是表达式，能直接返回值，并使用严格比较：

```php
$text = match ($status) {
    0 => '待支付',
    1 => '已支付',
    2 => '已取消',
    default => '未知状态',
};
```

#### nullsafe 操作符 `?->`

```php
$name = $user?->profile?->name;
```

链路中出现 `null` 时返回 `null`，避免继续访问属性。

#### 命名参数

```php
$user = createUser(
    name: 'Tom',
    age: 18,
    isVip: true,
);
```

#### Enum 与 readonly

```php
enum OrderStatus: int
{
    case Pending = 0;
    case Paid = 1;
}

final class User
{
    public function __construct(
        public readonly string $name,
    ) {}
}
```

课程只做了基础认识：Enum 表达有限状态集合，readonly 防止属性初始化后被重新赋值。

### 3.6 输出与调试怎么选

| 工具 | 适用场景 | 特点 |
|---|---|---|
| `echo` | 正常输出字符串、数字、JSON 字符串 | 不显示类型 |
| `print_r` | 快速查看数组或对象结构 | 比较清爽 |
| `var_dump` | 查看变量值和精确类型 | 信息最详细 |

```php
echo $user['name'];
print_r($user);
var_dump($user);
```

不要直接 `echo $array`，否则可能出现 `Array to string conversion`。

### 3.7 高频内置函数

#### 字符串

- `trim()`：去掉两端空白；
- `explode()` / `implode()`：字符串与数组互转；
- `str_replace()`：替换；
- `str_contains()`：判断包含；
- `strlen()`：字节长度；
- `mb_strlen()`：多字节字符数量；
- `strpos()`：查找位置，必须用 `=== false` 判断失败。

```php
if (strpos('abc', 'a') !== false) {
    echo 'found';
}
```

#### 数组

- `count()`、`array_map()`、`array_filter()`、`array_reduce()`；
- `array_keys()`、`array_values()`、`array_column()`；
- `in_array()`、`array_key_exists()`、`array_merge()`；
- `array_slice()`、`sort()`、`usort()`。

#### 类型与存在性

```php
$age = 0;

isset($age); // true：变量已设置且不是 null
empty($age); // true：0 被视为“空”
```

`isset` 与 `empty` 不能机械互换；尤其要留意 `0`、`"0"`、空字符串、`false` 和空数组。

## 4. OOP 核心（Day 2）

### 4.1 类、对象、构造函数

```php
<?php

declare(strict_types=1);

class User
{
    public function __construct(
        private string $name,
        private int $age,
    ) {}

    public function profile(): string
    {
        return $this->name . ' is ' . $this->age;
    }
}

$user = new User('Tom', 18);
echo $user->profile();
```

这里使用了 PHP 8 构造函数属性提升，把“属性声明”和“构造时赋值”合并。

### 4.2 可见性与封装

| 关键字 | 当前类 | 子类 | 类外部 |
|---|---:|---:|---:|
| `public` | 可以 | 可以 | 可以 |
| `protected` | 可以 | 可以 | 不可以 |
| `private` | 可以 | 不可以 | 不可以 |

不要把所有状态都设为 `public`。例如订单状态应通过 `pay()` 等业务方法改变，而不是让外部任意写入 `999`。

### 4.3 `$this`、`self`、`static`、`->`、`::`

| 写法 | 含义 |
|---|---|
| `$this->name` | 当前对象的普通属性 |
| `$this->run()` | 当前对象的普通方法 |
| `self::$instance` | 当前定义类的静态属性 |
| `self::create()` | 当前定义类的静态方法 |
| `User::ROLE` | User 类常量 |
| `User::instance()` | 调用 User 的静态方法 |
| `static::class` | 运行时调用类，支持后期静态绑定 |

容易写错的地方：

```php
$this->name;     // 对：对象属性名处不再写 $
self::$instance; // 对：静态属性名要写 $
User::ROLE;      // 对：类常量不写 $
```

`BaseService::instance()` 中使用 `static::class` 而非 `self::class`，使 `UserService::instance()` 能注册并获取 `UserService` 自己的实例。

### 4.4 继承、重写与多态

```php
class Animal
{
    public function speak(): string
    {
        return 'some sound';
    }
}

class Dog extends Animal
{
    public function speak(): string
    {
        return 'wang wang';
    }
}

class Cat extends Animal
{
    public function speak(): string
    {
        return 'miao miao';
    }
}

function printSound(Animal $animal): void
{
    echo $animal->speak() . PHP_EOL;
}

printSound(new Dog());
printSound(new Cat());
```

同一个 `Animal` 参数可以接收不同子类，调用相同方法时执行各自实现，这就是多态。

### 4.5 Interface

Interface 定义能力契约，不关心内部实现：

```php
interface PaymentInterface
{
    public function pay(int $amount): bool;
}

class StripePayment implements PaymentInterface
{
    public function pay(int $amount): bool
    {
        return true;
    }
}

function checkout(PaymentInterface $payment): bool
{
    return $payment->pay(100);
}
```

PHP interface 在运行时存在并约束实现类；TypeScript interface 主要在编译期发挥作用，编译后通常消失。

### 4.6 Abstract Class

抽象类可以同时提供公共实现和抽象约束：

```php
abstract class BaseController
{
    protected function success(array $data): array
    {
        return ['code' => 0, 'data' => $data];
    }

    abstract public function handle(): array;
}
```

选择口诀：

- 只定义能力契约：`interface`；
- 既提供公共逻辑，又要求子类补齐部分实现：`abstract class`；
- 表达明确的“是一种”关系：`extends`。

### 4.7 单例的基础认识

```php
class ConfigService
{
    private static ?ConfigService $instance = null;

    private function __construct()
    {
    }

    public static function instance(): ConfigService
    {
        if (self::$instance === null) {
            self::$instance = new self();
        }

        return self::$instance;
    }
}
```

课程源码中的 `BaseService`、`BaseRepository` 不是手写静态属性缓存，而是借助 Yii 依赖注入容器注册和获取单例。两者目标相似，实现机制不同。

## 5. Composer、namespace 与自动加载（Day 1、Day 3）

### 5.1 Composer 负责什么

```text
PHP                    ≈ Node.js runtime
Composer               ≈ npm / pnpm
composer.json           ≈ package.json
composer.lock           ≈ 锁定依赖版本的 lock 文件
vendor/                 ≈ node_modules/
composer install        ≈ npm install
vendor/autoload.php     ≈ 自动加载入口
```

最小 `composer.json`：

```json
{
  "autoload": {
    "psr-4": {
      "App\\": "src/"
    }
  }
}
```

修改 autoload 配置后执行：

```bash
composer dump-autoload
```

### 5.2 namespace 与 `use`

```php
<?php

declare(strict_types=1);

namespace App\Services;

use App\Repositories\UserRepository;

class UserService
{
}
```

常见文件顺序：

```text
<?php
declare(strict_types=1);
namespace ...
use ...
class ...
```

准确区分：

- `namespace`：声明当前类的完整命名空间；
- `use`：导入完整类名并建立短别名；
- Composer autoloader：根据类名查找并加载文件；
- `use ... as ...`：解决两个类短名相同的问题。

```php
use App\Pay\Services\UserService as PayUserService;
use App\Order\Services\UserService as OrderUserService;
```

### 5.3 PSR-4 映射算法

若 `App\` 映射到 `src/`：

```text
App\Services\UserService
→ 去掉前缀 App\
→ Services\UserService
→ 反斜杠替换为目录分隔
→ src/Services/UserService.php
```

| 完整类名 | 文件路径 |
|---|---|
| `App\User` | `src/User.php` |
| `App\Services\UserService` | `src/Services/UserService.php` |
| `App\Repositories\UserRepository` | `src/Repositories/UserRepository.php` |
| `App\Controllers\OrderController` | `src/Controllers/OrderController.php` |

### 5.4 mall-core 的真实差异

课程通用示例使用 Composer PSR-4，但源码阅读发现 `mall-core/composer.json` 的 autoload 只有：

```json
{
  "autoload": {
    "files": [
      "common/libraries/App/fun_helpers.php"
    ]
  }
}
```

这意味着：

- `autoload.files` 会无条件引入 helper 文件，使其中的全局函数可直接使用；
- 该项目没有在 Composer 中配置 `common\` 的 PSR-4；
- `common\BaseRepository` 等类由 Yii2 alias/autoloader 映射；
- 例如 `@common` → `common/`，所以 `common\BaseRepository` → `common/BaseRepository.php`。

因此，读真实项目时应按以下顺序排查：

1. 查 `composer.json` 的 `autoload.psr-4`；
2. 查 `autoload.files`、`classmap` 等其他 Composer 规则；
3. 如果仍找不到，检查框架自身 alias/autoloader；
4. 最终以项目实际配置为准，不把教程示例强套到源码。

## 6. Trait、异常与分层（Day 4）

### 6.1 Trait：横向复用能力

```php
trait LogTrait
{
    public function log(string $message): void
    {
        echo '[LOG] ' . $message . PHP_EOL;
    }
}

class UserService
{
    use LogTrait;
}
```

Trait 与继承：

| 对比项 | 继承 | Trait |
|---|---|---|
| 表达关系 | is-a，是一种 | 拥有某种能力 |
| 数量 | PHP 单继承 | 可使用多个 Trait |
| 目的 | 建立父子层次 | 复用横切能力 |

两个 Trait 出现同名方法时，可用 `insteadof` 和 `as` 解决：

```php
use A, B {
    A::hello insteadof B;
    B::hello as helloFromB;
}
```

### 6.2 Exception：异常流程

```php
function divide(int $a, int $b): float
{
    if ($b === 0) {
        throw new InvalidArgumentException('除数不能为 0');
    }

    return $a / $b;
}

try {
    echo divide(10, 0);
} catch (InvalidArgumentException $e) {
    echo $e->getMessage();
} finally {
    echo PHP_EOL . '计算结束';
}
```

异常对象常用信息：

- `$e->getMessage()`：错误消息；
- `$e->getCode()`：错误码；
- `$e->getFile()` / `$e->getLine()`：发生位置；
- `$e->getTraceAsString()`：调用链。

`finally` 适合关闭资源、清理状态和记录结束日志。不要为了“看起来安全”而在每一层重复捕获；应在能够处理、转换或统一记录异常的边界捕获。

### 6.3 Controller、Service、Repository

| 层 | 核心问题 | 典型职责 |
|---|---|---|
| Controller | 请求是什么 | 路由参数、请求体、调用 Service、返回响应 |
| Service | 业务怎么做 | 业务校验、流程编排、状态判断 |
| Repository | 数据怎么取和存 | 查询、保存、更新、隔离 ORM/SQL |
| Model/ORM | 数据如何映射 | 表结构映射、执行数据库操作 |

一句话记忆：

```text
Controller 接请求，Service 管业务，Repository 管数据访问。
```

Prisma 是 Node.js 生态的 ORM/数据库工具，不是业务层。类比链路为：

```text
Controller → Service → Repository → Prisma → Database
```

PHP 项目中则可能是 Repository 调用 ActiveRecord、ORM 或 SQL。

## 7. PSR-12 与现代 PHP 书写习惯（Day 5）

### 7.1 PSR-4 和 PSR-12 不要混淆

| 规范 | 解决的问题 |
|---|---|
| PSR-4 | 命名空间如何映射到文件路径 |
| PSR-12 | PHP 代码如何统一排版 |

### 7.2 本周应养成的格式习惯

```php
<?php

declare(strict_types=1);

namespace App\Services;

use App\Repositories\UserRepository;

class UserService
{
    public function __construct(
        private UserRepository $repository,
    ) {}

    public function findName(int $id): string
    {
        return 'User#' . $id;
    }
}
```

复习重点：

- 使用 4 个空格缩进；
- 类和方法的左大括号换行；
- 方法明确写可见性；
- 参数、属性和返回值尽量声明类型；
- 类名使用 PascalCase，方法通常使用 camelCase；
- 接口常以 `Interface` 结尾，Trait 常以 `Trait` 结尾；
- 类常量使用 `UPPER_SNAKE_CASE`；
- 纯 PHP 文件不写结束标签。

真实老项目可能没有完整类型声明或使用历史命名风格。阅读时先理解现状；新增代码尽量保持团队约定，不要脱离改造计划直接全仓格式化。

## 8. JSON、HTTP 与 Todo REST API（Day 5、Day 6）

### 8.1 REST 资源与方法

| 操作 | 方法 | 路径 | 常用成功状态码 |
|---|---|---|---|
| 获取列表 | GET | `/todos` | 200 |
| 获取详情 | GET | `/todos/{id}` | 200 |
| 创建 | POST | `/todos` | 201 |
| 更新 | PUT | `/todos/{id}` | 200 |
| 删除 | DELETE | `/todos/{id}` | 200 |

常见错误状态码：

- `400 Bad Request`：请求参数错误；
- `404 Not Found`：路由或资源不存在；
- `500 Server Error`：未妥善处理的服务端异常。

### 8.2 PHP 读取请求

```php
$method = $_SERVER['REQUEST_METHOD'];
$path = parse_url($_SERVER['REQUEST_URI'], PHP_URL_PATH) ?: '/';
$rawBody = file_get_contents('php://input') ?: '{}';
$payload = json_decode($rawBody, true) ?: [];
```

这里的练习写法简洁，但有一个限制：无效 JSON 和合法 JSON `null` 都可能落到空数组，无法给出精确错误。更稳妥的复习方向是使用 `JSON_THROW_ON_ERROR` 捕获 `JsonException`。

### 8.3 统一 JSON 响应

```php
final class Response
{
    public static function json(
        int $httpStatus,
        int $code,
        string $message,
        mixed $data = null,
    ): void {
        http_response_code($httpStatus);
        header('Content-Type: application/json; charset=utf-8');

        echo json_encode(
            [
                'code' => $code,
                'message' => $message,
                'data' => $data,
            ],
            JSON_UNESCAPED_UNICODE | JSON_THROW_ON_ERROR,
        );
    }
}
```

统一结构：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

HTTP 状态码和业务 `code` 是两个维度：前者描述 HTTP 请求结果，后者是应用内部约定。

### 8.4 `json_encode` / `json_decode`

```text
PHP 变量  --json_encode--> JSON 字符串
JSON 字符串 --json_decode--> PHP 变量
```

```php
$json = json_encode(
    ['message' => '成功'],
    JSON_UNESCAPED_UNICODE | JSON_THROW_ON_ERROR,
);

$data = json_decode($json, true, 512, JSON_THROW_ON_ERROR);
```

数组编码规则是高频坑：

- 字符串键关联数组 → JSON 对象 `{}`；
- 从 0 开始的连续整数键 → JSON 数组 `[]`；
- 不连续整数键 → JSON 对象 `{}`；
- 列表过滤后用 `array_values()` 重排键。

```php
$doneTodos = array_values(
    array_filter($todos, fn (array $todo): bool => $todo['done']),
);
```

编码失败可能源于无效 UTF-8、循环引用或嵌套过深。可用：

- `json_last_error_msg()` 紧跟 JSON 操作检查；
- 更推荐 `JSON_THROW_ON_ERROR` 配合 `try/catch`。

## 9. 源码阅读收获

### 9.1 `BaseService`

`common\BaseService` 是服务层基类，主要沉淀：

- 使用 Yii 容器的 `instance()` 单例入口；
- `retArray()`、`returnFormat()`、`returnSuccess()`、`returnError()` 等统一返回结构；
- `contextInit()` 对站点、语言、币种、token、地区、经纬度等公共参数进行初始化；
- `setCommonParams()` 与 `CommonParamsBuilder` 管理公共参数；
- `getMillisecond()` 生成毫秒级时间戳。

关键代码思想：

```php
public static function instance()
{
    $container = Yii::$container;

    if (!$container->hasSingleton(static::class)) {
        $container->setSingleton(static::class);
    }

    return $container->get(static::class);
}
```

收获：

- 企业基类的价值是沉淀重复能力，而不是承载所有业务；
- `static::class` 让子类继承 `instance()` 后获取子类实例；
- 源码是老项目风格，缺少本周推荐的完整类型声明，阅读时要区分“理解遗留代码”和“新代码质量目标”；
- `retArray()` 与 `returnFormat()` 功能近似，提示真实项目可能存在历史重复和兼容成本。

### 9.2 `BaseRepository`

`common\BaseRepository` 是抽象数据访问基类，包含：

- `DEL_FLAG_0`、`DEL_FLAG_1` 软删除标志；
- `instance($useSlave = false)` 容器单例入口；
- `getDb()` 和 `getSlaveDb()` 主从连接选择；
- 抽象 `getConnection()`，要求具体 Repository 指定连接。

它不能直接实例化，具体业务 Repository 必须继承并实现抽象方法。

### 9.3 主库、从库与读写分离

```text
写请求 → 主库 → 复制 → 从库
读请求 ─────────────→ 从库
```

- 主库负责写，也可以读；
- 从库主要承担读压力；
- 主从复制存在延迟；
- “刚写完马上读取”的强一致场景通常应读主库；
- `$useSlave` 是可控开关，不应把所有查询无条件发往从库。

### 9.4 源码阅读方法

遇到一个陌生类，按下面顺序阅读：

1. 看文件路径、namespace、class 名；
2. 看 `use` 依赖、`extends`、`implements`、Trait；
3. 看常量和属性，判断它保存什么状态；
4. 先列 public/protected/abstract 方法，不急着钻细节；
5. 找入口方法，例如 `instance()`；
6. 判断它属于 Controller、Service、Repository 还是工具类；
7. 回到 `composer.json` 或框架 alias，确认类如何被加载；
8. 用一句话总结“它给谁提供什么公共能力”。

## 10. 容易混淆点

### 10.1 `use` 与自动加载

错误理解：`use` 会把文件加载进来。  
正确理解：`use` 主要建立类名别名；Composer 或框架 autoloader 才负责加载文件。

### 10.2 PSR-4 与 PSR-12

- PSR-4：类在哪里；
- PSR-12：代码怎么排版。

### 10.3 Trait、Interface、Abstract Class

- Trait：复用实现；
- Interface：规定契约；
- Abstract Class：复用部分实现并规定剩余契约。

### 10.4 `$this`、`self`、`static`

- `$this`：当前对象；
- `self`：代码定义所在的当前类；
- `static`：运行时调用类，常用于继承场景的后期静态绑定。

### 10.5 `->` 与 `::`

- `->`：访问对象成员；
- `::`：访问类级静态成员、类常量或类名。

### 10.6 Service 与 Repository

- Service 不应退化成 SQL/ORM 查询集合；
- Repository 不应知道完整业务流程；
- Todo 练习没有 Repository，是因为它用内存数组模拟数据，没有接数据库。

### 10.7 HTTP 状态码与业务码

`http_response_code(404)` 是协议层结果；JSON 中的 `"code": 1` 是应用层约定，不能互相替代。

### 10.8 PHP 数组与 JSON

PHP 数组既可像列表，也可像字典。JSON 输出是 `[]` 还是 `{}` 取决于键是否为从 0 开始的连续整数。

### 10.9 `isset`、`empty` 与 `null`

- `isset($x)`：已设置且不为 `null`；
- `empty($x)`：很多值都会被当成空，包括 `0`；
- 判断布尔字段或数字字段时，不能只用 `empty()`。

### 10.10 通用示例与真实项目

通用 demo 使用 `App\` + Composer PSR-4；`mall-core` 实际使用 Yii2 alias 映射 `common\` 等命名空间。核心思想相似，但加载来源不同。

## 11. 常见错误与排查

### 11.1 `php: command not found`

```bash
which php
php -v
```

若未安装，可在 macOS 使用 Homebrew 安装 PHP；若已安装则检查 PATH。

### 11.2 `composer: command not found`

```bash
which composer
composer -V
```

确认 Composer 已安装并位于 PATH。还可运行：

```bash
composer diagnose
```

### 11.3 `Class "App\..." not found`

依次检查：

1. 是否引入 `vendor/autoload.php`；
2. `composer.json` 的 PSR-4 前缀是否正确；
3. namespace、目录、类名和文件名大小写是否一致；
4. 修改 autoload 后是否执行 `composer dump-autoload`；
5. 项目是否改由框架 alias/autoloader 加载。

### 11.4 `TypeError`

检查：

- 传入参数的真实类型，可用 `var_dump()`；
- 是否把 HTTP 字符串直接传给 `int`、`bool` 参数；
- 返回值是否符合声明；
- `strict_types` 是否开启；
- 是否缺少输入校验或显式转换。

### 11.5 `Array to string conversion`

不要 `echo $array`。调试用 `print_r()` / `var_dump()`，接口输出用 `json_encode()`。

### 11.6 `Cannot modify header information` / `headers already sent`

`header()` 和 `http_response_code()` 必须在任何输出前调用。检查：

- 文件开头是否有 BOM；
- `<?php` 前是否有空格；
- 是否提前 `echo`、`print_r` 或 `var_dump`；
- 纯 PHP 文件是否多写了 `?>` 并在其后产生空白。

### 11.7 JSON 解析失败

练习阶段可立即检查：

```php
$data = json_decode($rawBody, true);

if (json_last_error() !== JSON_ERROR_NONE) {
    echo json_last_error_msg();
}
```

现代写法可使用 `JSON_THROW_ON_ERROR` 并捕获 `JsonException`。

### 11.8 列表 JSON 意外变成对象

`array_filter()` 保留原键，过滤后执行：

```php
$list = array_values(array_filter($list, $callback));
```

### 11.9 新建 Todo 后下一次请求找不到

这是练习实现的预期限制：每个 HTTP 请求都会重新创建 `TodoService`，内存数组被重置。要持久化必须接数据库或其他存储。

### 11.10 写后立即从从库读不到数据

这是主从复制延迟。对写后立即读等强一致场景，改为读主库。

## 13. 高频面试 / 自测题（附简短答案）

### 13.1 PHP 与类型

1. **`strict_types=1` 有什么用？**  
   让标量类型检查更严格，减少隐式类型转换；仍需配合类型声明和输入校验。

2. **PHP 的 `array` 与 JS Array 有何不同？**  
   PHP `array` 既可表示连续列表，也可表示键值字典。

3. **`mixed` 适合什么场景？**  
   确实需要接受任意类型的通用边界，例如统一响应或序列化；不应替代明确业务类型。

4. **`echo`、`print_r`、`var_dump` 如何选择？**  
   `echo` 输出简单值，`print_r` 看数组结构，`var_dump` 看类型和值。

### 13.2 Composer 与命名空间

5. **`vendor/autoload.php` 是什么？**  
   Composer 生成的自动加载入口，引入后可按配置查找类文件。

6. **`use` 会直接加载文件吗？**  
   不会。它建立类名别名，文件加载由 autoloader 完成。

7. **PSR-4 是什么？**  
   命名空间前缀到目录的自动加载规范。

8. **`App\Services\UserService` 如何映射到文件？**  
   若 `App\` → `src/`，则映射为 `src/Services/UserService.php`。

9. **修改 autoload 配置后做什么？**  
   执行 `composer dump-autoload`。

10. **为什么 mall-core 没有 PSR-4 仍能加载 `common\BaseRepository`？**  
    Yii2 alias/autoloader 将 `@common` 映射到 `common/`。

### 13.3 OOP

11. **继承和多态是什么关系？**  
    子类重写父类方法后，父类类型可接收不同子类并表现出不同行为，形成多态。

12. **Interface 与 Abstract Class 的区别？**  
    Interface 主要定义契约；抽象类可包含公共实现和抽象方法。

13. **Trait 与继承的区别？**  
    继承表达 is-a；Trait 横向复用能力，且一个类可以使用多个 Trait。

14. **`$this->name` 与 `self::$name` 的区别？**  
    前者访问当前对象属性，后者访问当前类静态属性。

15. **`static::class` 相比 `self::class` 的意义？**  
    `static::class` 支持后期静态绑定，子类调用时指向子类。

16. **单例模式解决什么问题？**  
    统一获取并复用同一个实例；课程源码通过 Yii 容器实现。

### 13.4 分层与异常

17. **Service 和 Repository 如何分工？**  
    Service 管业务规则和流程，Repository 管数据查询与持久化。

18. **`finally` 什么时候执行？**  
    正常完成或发生异常后通常都会执行，适合清理资源。

19. **为什么要统一错误响应？**  
    让客户端稳定解析，减少各接口格式不一致的适配成本。

20. **BaseService/BaseRepository 的价值是什么？**  
    沉淀同类组件的公共能力，减少重复并统一调用方式。

### 13.5 REST 与 JSON

21. **`POST /todos` 成功常用什么状态码？**  
    `201 Created`。

22. **PHP 如何读取请求方法和 JSON body？**  
    方法读 `$_SERVER['REQUEST_METHOD']`，body 读 `php://input` 后 `json_decode()`。

23. **为什么 PHP 列表有时编码成 `{}`？**  
    整数键不是从 0 开始连续；用 `array_values()` 重排。

24. **`JSON_UNESCAPED_UNICODE` 做什么？**  
    让中文在 JSON 中保持可读，不转成 `\uXXXX`。

25. **HTTP 状态码和业务 code 有什么区别？**  
    HTTP 状态码描述协议层结果；业务 code 表达应用内部结果。

26. **练习版 Todo API 为什么不能持久保存新增数据？**  
    数据只在单次请求新建的 Service 对象内，下一次请求会重新初始化。

### 13.6 源码

27. **BaseRepository 为什么是抽象类？**  
    它提供通用连接与单例能力，并要求子类实现 `getConnection()`。

28. **为什么写后立即读可能必须走主库？**  
    主从复制有延迟，从库可能尚未收到最新数据。

29. **`autoload.files` 与 PSR-4 的区别？**  
    `files` 无条件引入指定文件；PSR-4 在使用类时按命名空间映射加载文件。

30. **进入 Yii2 学习前应记住的请求链路是什么？**  
    请求 → `index.php` → Application → Module → Controller → action → Response。
